1. What is ESS-0000757?

The European Spallation Source (ESS) is a neutron research facility being built in Lund, Sweden. It is one of the largest science infrastructure projects in Europe, involving 11 partner institutes across 9 countries.

ESS uses EPICS (Experimental Physics and Industrial Control System) to control thousands of devices — from temperature sensors to accelerator magnets. Every device exposes its measurements and settings through Process Variables (PVs).

When hundreds of engineers across multiple countries create PV names independently, chaos follows. Operators in the control room need to find the right PV instantly. Alarm systems need consistent names. Archiving needs predictable patterns.

ESS-0000757 is the official naming convention document that solves this problem. Revision 10 defines exactly how every PV name at ESS must be structured. It is maintained by Jorick Lochet (ESS Naming Coordinator) and published on CHESS, the ESS document management system.

pvValidator is the tool that enforces these rules automatically. Instead of reading 33 pages of rules, you paste your PV names into pvValidator and it tells you what needs to change.

2. Anatomy of a PV Name

Click on any segment to learn what it means:

DTL - 010 : EMR - TT - 001 : Temperature
System (DTL) — Where is this device? DTL = Drift Tube Linac, one of the accelerator sections. Systems are registered in the ESS Naming Service. 1-6 characters, must start with a letter.
Subsystem (010) — Which section? 010 = section 10 of the DTL. Subsystems can start with digits (e.g., DTL-010 for section numbers). 1-6 characters.
Discipline (EMR) — What engineering discipline? EMR = Electrical Measurement & Regulation. Other examples: Vac (Vacuum), Cryo (Cryogenics), Ctrl (Control). 1-6 characters, must start with a letter.
Device (TT) — What type of device? TT = Temperature Transmitter. Other examples: PT (Pressure Transmitter), VG (Vacuum Gauge). 1-6 characters, must start with a letter.
Index (001) — Which instance? 001 = the first temperature transmitter. Use Scientific style (1-4 digits) or P&ID style (3 digits + optional lowercase, e.g., 002a for redundant devices).
Property (Temperature) — What value does this PV represent? This is the part you choose as an integrator. Must start with an uppercase letter, use PascalCase for multi-word names (MassFlow, not massflow). Max 25 characters.
The three colons rule: Every PV name has exactly 2 colons, creating 3 parts:
SystemPart : DevicePart : Property

3. The Four Formats

Not every PV has all segments. ESS defines four valid formats:

Do you have a specific device?
Yes → Do you have a subsystem?
Yes → Sys-Sub:Dis-Dev-Idx:Property (full format)
No → Sys:Dis-Dev-Idx:Property (no subsystem)
No (system-wide value) → Do you have a subsystem?
Yes → Sys-Sub::Property (high-level with subsystem)
No → Sys::Property (high-level)
FormatExampleWhen to use
FullDTL-010:EMR-TT-001:TemperatureMost common. A specific device in a specific subsystem.
No subsystemISrc:Vac-VG-001:PressureSystems that don't have subsections.
High-level (sub)DTL-010::TemperatureAggregated value for a whole subsystem.
High-levelTgt::PowerConsumptionSystem-wide value (no device, no subsystem).
Note the double colon in high-level PVs: Sys::Property. The device section is empty, but both colons must be present.
